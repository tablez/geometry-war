# Geometry War（几何尖塔）游戏设计文档

## 游戏概述

**类型：** Roguelike 卡牌爬塔游戏  
**平台：** 微信小程序 / H5  
**核心玩法：** 卡牌战斗 + 爬塔 + 角色养成  
**参考游戏：** 杀戮尖塔 (Slay the Spire)

---

## 核心循环

```
选择职业 → 爬塔 → 遭遇敌人/事件 → 卡牌战斗 → 获得奖励 → 继续爬塔 → Boss战 → 通关/失败
```

---

## 职业系统

### 战士（已完成）
- **定位：** 高生命、护甲流派、卖血流派
- **初始卡组：** 4张打击 + 4张防御 + 1张头槌打击 + 1张耸肩无视
- **特色机制：** 卖血换能量、护甲叠加、力量增益

### 计划职业（MVP后）
- **法师：** 高能量、法术伤害、群体攻击
- **刺客：** 高暴击、连击、快速过牌
- **牧师：** 治疗、防御、辅助增益

---

## 卡牌系统

### 卡牌类型
- **攻击牌：** 造成伤害
- **技能牌：** 护甲、治疗、抽牌
- **能力牌：** 持续效果（力量、敏捷等）
- **诅咒牌：** 负面效果

### 卡牌稀有度
- **基础牌：** 初始卡组
- **普通牌：** 常见奖励
- **稀有牌：** 强力效果
- **传说牌：** 改变流派

---

## 战斗系统

### 回合流程
1. 抽牌阶段（默认抽5张）
2. 玩家回合（出牌、使用能量）
3. 敌人回合（敌人行动）
4. 结束回合（弃牌、敌人攻击）

### 能量系统
- 每回合恢复3点能量
- 卡牌消耗1-3点能量
- 可通过卡牌/遗物增加能量

### 状态效果
- **力量：** 增加攻击伤害
- **敏捷：** 增加护甲值
- **易伤：** 受到更多伤害
- **虚弱：** 造成更少伤害
- **中毒：** 每回合受到伤害

---

## 爬塔系统

### 楼层类型
- **普通战斗：** 标准敌人
- **精英战斗：** 强力敌人，更好奖励
- **Boss战：** 每10层一个Boss
- **休息处：** 回复生命或升级卡牌
- **商店：** 购买卡牌和遗物
- **事件：** 随机事件

### 敌人设计
- **史莱姆：** 基础敌人
- **哥布林：** 中等敌人
- **骷髅兵：** 高伤害
- **Boss：** 特殊机制

## 技术架构

### 技术栈
- **框架：** uni-app + Vue3
- **状态管理：** Pinia (gameStore.js)
- **样式：** CSS/SCSS

### 文件结构
```
pages/
├── index/          # 首页/职业选择
├── game/           # 战斗页面
├── map/            # 爬塔地图
├── rest/           # 休息处
├── shop/           # 商店
└── reward/         # 奖励页面

static/
├── cards-complete.js    # 卡牌数据
├── game-config.js       # 游戏配置
└── tower-config.js      # 爬塔配置

store/
└── gameStore.js    # 游戏状态管理
```

## 设计原则

1. **简单上手：** 核心机制清晰易懂
2. **深度策略：** 卡牌组合、流派构建
3. **随机性：** 每局不同体验
4. **渐进难度：** 从简单到挑战
